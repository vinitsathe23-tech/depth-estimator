System Design Overview (SWE2)

The system follows a modular pipeline architecture with clear separation of concerns:

1. Input Layer

FrameCapture: Unified interface to video sources (webcam or file)

  • Parse and validate video sources
  • Handle resolution and frame rate
  • Error handling for unavailable sources

2. Processing Pipeline

Perception Orchestrator: Reusable frame-processing coordinator for this app and future apps

Depth Estimator: Multiple backends (Pseudo-Depth, Depth Anything v2)

Object Detector[]: One or more backends (Depth-Blob, YOLO)

Calibration Engine: One-point calibration for distance mapping

Position Estimator: 3D coordinate calculation

3. State Management

Configuration Manager: CLI args and config files

Calibration State: Persistent calibration parameters

Frame Buffer: Pipeline buffering

4. Output Layer

Visualizer: 3-panel real-time display

Data Exporter: Save frames and position data

Component Details (SWE3 - Detailed Design)

Perception Orchestrator

Purpose: Reusable frame-processing coordinator that returns frame, depth, detections, primary detection, and timing metadata.

class PerceptionResult:
    frame: np.ndarray
    depth: np.ndarray
    detections: list[DetectedObject]
    primary_detection: DetectedObject | None
    timings_ms: dict[str, float]

Depth Estimator (Abstract)

Backend Latency Accuracy GPU Required
PseudoDepth - OpenCV edge/corner detection ~5ms Low (baseline) No
DepthAnythingV2 - AI model (HuggingFace) ~45-60ms (RTX 4060 Ti) High (±10%) Yes (~2GB VRAM)

Object Detector (Abstract)

Backend Latency Method GPU Required
DepthBlobDetector - Contour-based ~3ms Finds nearest depth discontinuity No
YOLOPhoneDetector - Ultralytics YOLO v8 CPU: ~15-25ms | GPU: ~5-10ms Class filtering (cell phone) Optional

Detection Object Structure

class DetectedObject:
    bbox: tuple[int, int, int, int]  # (x1, y1, x2, y2)
    center: tuple[int, int]          # (cx, cy)
    relative_depth: float            # 0-1 normalized
    distance_m: float | None         # calibrated distance in meters
    x_m: float | None                # lateral position
    label: str                       # object label
    confidence: float | None         # detection confidence score

Calibration Engine

Calibration Model:

distance_m = a * relative_depth + b

Where:
- relative_depth ∈ [0, 1]
- User provides: known_distance_m at one known relative_depth value
- Solver estimates: a, b to fit calibration point

Coordinate System

  • Z-axis: Forward distance (depth) in meters
  • X-axis: Lateral offset in meters (positive = right)
  • Calculation: x = (center_x - image_center_x) * z * tan(half_fov) / image_width

Design Patterns Applied

🔄 Strategy Pattern

Usage: DepthEstimator and ObjectDetector abstract classes consumed by PerceptionOrchestrator

Benefit: Pluggable backends without modifying core logic

Example: Swap between PseudoDepth and DepthAnythingV2 via configuration

🏭 Factory Pattern

Usage: Depth estimator and detector creation helpers

Benefit: Centralized instantiation of backends based on configuration

Example: Factory selects correct backend based on CLI argument

🎛️ State Pattern

Usage: CalibrationEngine manages calibration state

Benefit: Encapsulates calibration logic and state transitions

Example: Transition from uncalibrated → calibrated state

📡 Observer Pattern (Future)

Usage: Event-driven updates for real-time data

Benefit: Loose coupling between components

Example: Subscribe to detection events for logging/export
